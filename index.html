Testes Unitários

O que é um Teste Unitário?

Um teste unitário é uma técnica de desenvolvimento de software em que pequenas partes do código (geralmente chamadas de unidades) são testadas isoladamente para garantir que funcionam como esperado.

Características principais de um Teste Unitário:

  • Isolamento: Testa apenas uma funcionalidade ou componente específico.
  • Automação: Deve ser possível rodar o teste automaticamente, sem intervenção manual.
  • Previsibilidade: Um teste unitário bem escrito sempre retorna o mesmo resultado para os mesmos dados de entrada.
  • Foco: Verifica apenas o comportamento esperado de uma unidade do sistema.

Etapas do teste Unitário

  • Preparação: Configura o ambiente para o teste, criando dados ou simulando componentes.
  • Execução: Chama a unidade de código sendo testada, com entradas específicas.
  • Validação: Compara o resultado obtido com o esperado.
